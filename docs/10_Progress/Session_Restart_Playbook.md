# Session Restart Playbook

Цель: быстро восстановить контекст проекта AniForge на новом устройстве или в новой сессии с нулевой памятью чата.

## Когда использовать
- Новый компьютер.
- Новый чат/сессия (потерян предыдущий контекст).
- Длинный перерыв в работе.

## Шаг 1. Открыть vault
1. Открыть Obsidian.
2. Выбрать папку `docs` как vault.
3. Открыть заметки:
- [[00_Home]]
- [[10_Progress/As_Built_Status]]
- [[02_Product/Collaboration_Protocol]]
- [[01_Roadmap/Phase_1_MVP]]

## Шаг 2. Быстрый self-check (2-3 минуты)
Проверить и зафиксировать:
- Какой последний завершенный коммит?
- Какой пункт roadmap закрыт?
- Какая одна следующая микро-задача?
- Какие ограничения среды сейчас актуальны (docker/db/доступы)?

## Шаг 3. Шаблон первого сообщения в новый чат
Скопировать и отправить:

```text
Контекст: AniForge. Работаем по правилу 1 пункт чеклиста = 1 задача = 1 коммит.
Твоя роль: архитектор-ментор, моя роль: исполнитель.
Источник истины: docs/ (Obsidian vault).
Прочитай и синхронизируйся по файлам:
- docs/00_Home.md
- docs/10_Progress/As_Built_Status.md
- docs/02_Product/Collaboration_Protocol.md
- docs/01_Roadmap/Phase_1_MVP.md
После синхронизации дай 1 следующую микро-задачу с DoD и командой проверки.
```

## Шаг 4. Формат выполнения микро-задачи
1. Получить задачу + DoD.
2. Выполнить локально.
3. Проверить сборку/тесты (минимум `pnpm build`, если применимо).
4. Один коммит.
5. Обновить docs статуса (если затронут roadmap/архитектура).

## Шаг 5. Что обновлять после каждой задачи
- [[10_Progress/As_Built_Status]]
- профильный документ направления (frontend/backend/devops/ai)
- при необходимости: [[01_Roadmap/Phase_1_MVP]] / [[01_Roadmap/Phase_2_Expansion]] / [[01_Roadmap/Phase_3_Startup]]

## Антипаттерны (чего избегать)
- Несколько задач в одном коммите.
- Изменения без фиксации DoD.
- Новая фича без обновления статуса в docs.
- Попытка решить блокер молча, без формулировки гипотезы.
